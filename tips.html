Every Browser behaves different so you probably will have different spritespin settings for some cases. Spritespin will only consume the options you provide and will not run different code for different browsers. Its up to you to find the optimal settings for your targeting browsers. Here is a collection of tips that may be helpful to accomplish this task.

Unstitched Images

Unstitched Images are the easiest way to get an animation running. They work well on almost all desktop browsers. However flickering animation could occur on some mobile devices and also in Opera. The solution is to use stitched images.

Stitched Images / Spritesheets

One way to stitch images is to put them all in a row. This is not recommended since it will slow down the Chrome Browser if you use large images. It's Better to pack them in a grid e.g. 36 frames in a 6x6 grid. In this case you have to set the framesX option to 6 or whatever number of frames you have per column.

iPhone limitation

If you stitch many frames together the image will probably exceed the maximum image size for iPhones. In this case the browser will scale down the image and use a low resolution copy which leads spritepsin to work not correctly. To solve this you have to specify the options resolutionX and resolutionY so the stitched image will be displayed in that resolution even if it has been scaled down.
